01
Resumen ejecutivo
Onyx es una plataforma empresarial de monitoreo, analítica e inteligencia de endpoints diseñada para organizaciones con flotas de más de 1,000 equipos Windows 11. Combina un agente nativo ultraligero instalado en cada computador con una arquitectura de datos e inteligencia artificial completamente hospedada en Google Cloud Platform.

La plataforma convierte el uso diario de los equipos en eficiencia, claridad y productividad medible. Opera con continuidad total, incluso sin conectividad, sincronizando automáticamente los datos al restaurarse la red.

El producto atiende una necesidad crítica en organizaciones medianas y grandes: la ausencia de visibilidad real sobre lo que ocurre en los equipos corporativos — desde el comportamiento de hardware hasta la productividad del colaborador. Onyx cierra esa brecha con datos en tiempo real, alertas inteligentes y herramientas de análisis accionables.

Problemas que resuelve
🔍

Visibilidad nula o fragmentada

No existe una fuente única de verdad sobre el estado real de la flota de equipos.

🔗

Sin correlación falla–usuario

Es imposible conectar una falla de hardware con el comportamiento específico del usuario que la causó.

📊

Sin indicadores de productividad

La productividad se gestiona por percepción, no por datos objetivos de uso de aplicaciones y tiempo.

🚨

Alertas reactivas

El equipo de TI reacciona ante incidentes en lugar de prevenirlos con datos anticipados.

🛡️

Riesgos de seguridad ocultos

Accesos fallidos, conexiones desde IPs extranjeras y equipos inactivos pasan desapercibidos.

📡

Datos dispersos

La información existe en múltiples herramientas desconectadas que impiden análisis de tendencias.

03
Arquitectura de la solución

La plataforma se organiza en cuatro capas desacopladas: el agente en el endpoint, la ingesta y streaming en GCP, el almacenamiento analítico, y la capa de presentación con agentes inteligentes.

CAPA 1 Endpoint — Windows 11 Go · SQLite · gRPC · TLS 1.3
  • Agente Go ultraligero (<1% CPU / <50 MB RAM) instalado como servicio Windows.
  • Módulos internos: HardwareMonitor, SoftwareInventory, ActivityTracker, NetworkMonitor, AlertEngine local.
  • Buffer local SQLite cifrado con AES-256-GCM; clave derivada del TPM del equipo.
  • Notificaciones nativas al usuario vía Windows Toast (WinRT), bilingüe (ES/EN).
  • Comunicación: gRPC sobre TLS 1.3 hacia Cloud Pub/Sub. Health-check periódico para detectar modo offline.
CAPA 2 Ingesta y Streaming — GCP Pub/Sub · Dataflow · Cloud Functions · Cloud Run
  • Cloud Pub/Sub: recibe eventos en tiempo real de toda la flota (métricas + alertas, topics separados).
  • Dataflow (Apache Beam): transformación, enriquecimiento con metadata del equipo, deduplicación de eventos offline.
  • Cloud Functions: evalúan reglas de alerta al vuelo; disparan agentes IA y crean tickets automáticos. [v1.1] learning-pattern-detector evalúa patrones de error sistémico.
  • Cloud Run: microservicios REST para configuración, comandos remotos y API del portal web. [v1.1] proy-anla-poc-whatsapp-bot (webhook WhatsApp + RBAC) y proy-anla-poc-chart-renderer (Plotly → PNG).
  • [v1.1] Cloud Scheduler: reentrenamiento quincenal Vertex AI + cálculo batch de KPIs personalizados.
CAPA 3 Almacenamiento y Analítica — GCP BigQuery · Spanner · Vertex AI · Cloud Storage
  • BigQuery: data warehouse principal con 15 tablas particionadas por fecha y clustering por device_id. Retención 1–7 años. [v1.1] +7 tablas nuevas (WhatsApp, aprendizaje, KPIs).
  • Cloud Spanner: estado en tiempo real de cada endpoint (última sincronización, última IP, estado actual).
  • Cloud Storage: logs históricos en Parquet, backups del agente, plantillas de Dataflow.
  • Vertex AI: modelos de detección de anomalías (Z-score), predicción de fallas (series temporales 7 días). [v1.1] Modelo KPI Advisor (Gemini Pro con grounding BigQuery) + pipeline de reentrenamiento.
  • BigQuery Graph / Spanner Graph: análisis de concentraciones para Graph Analytics.
CAPA 4 Presentación y Agentes React · Looker Studio · Vertex AI Agents · WebSockets
  • Portal web React + TypeScript en Cloud Run; dashboard con actualización en tiempo real vía WebSockets. [v1.1] +3 vistas: WhatsApp Admin, Aprendizaje Continuo, KPI Builder.
  • Looker Studio: dashboards ejecutivos conectados directamente a BigQuery para reportes sin código.
  • Vertex AI Agent Builder: 14 agentes con lógica de alertas proactivas, análisis de causa raíz, predicciones, aprendizaje continuo, canal WhatsApp y asesoría KPI.
  • API REST integrable con cualquier mesa de ayuda (ServiceNow, Jira, Freshdesk u otra vía webhook).
  • ChatScout: interfaz de lenguaje natural integrada al portal para consultas sobre datos de la flota.
  • [v1.1] Canal WhatsApp Business: analítica visual accesible desde WhatsApp con RBAC, OTP y audit log completo.
04
Funcionalidades clave

Las siguientes capacidades describen el comportamiento observable del sistema desde el punto de vista del negocio y el usuario final.

Monitoreo de hardware en tiempo real

Métricas de CPU, RAM, disco y red actualizadas cada 30 segundos. Dashboard con semáforo automático (normal / alertado / crítico / hibernando). Drilldown por equipo con top-10 procesos por consumo e identificación de causa raíz (navegador, antivirus, sistema, otro). Historial de picos de congestión exportable, filtrable por sede y área.

Inventario de software y cumplimiento de licencias

Catálogo completo de aplicaciones instaladas por equipo con estado de actualización. Agrupación por suite (Google Workspace, Microsoft 365, etc.). Visualización de uso real por licencia, licencias subutilizadas y software sin licencia. Alertas automáticas por versiones obsoletas con CVE asociado.

Productividad medible por colaborador y área

Clasificación automática de actividad en productiva (ofimática, comunicación, herramientas de negocio), no productiva (entretenimiento, redes sociales) e inactiva. Reporte semanal por usuario y por departamento. Nube de palabras con términos más frecuentes. Cálculo del costo económico del tiempo improductivo y valor de la hora efectiva.

Operación offline transparente

El agente detecta pérdida de conectividad automáticamente y almacena todos los eventos en buffer local cifrado. Al restaurarse la conexión, sincroniza respetando los timestamps originales. El portal refleja el estado correcto de cada equipo ("Offline desde [fecha]" → "Sincronizado [fecha]"). Sin pérdida de datos garantizada.

Alertas inteligentes y creación automática de tickets

10 agentes de IA con disparadores configurables. Si una condición crítica persiste más de 5 minutos tras la notificación al usuario, el sistema crea un ticket automático en la mesa de ayuda con evidencia adjunta (logs, métricas, proceso causante, historial). El usuario recibe el número de ticket. El agente de soporte recibe el contexto completo sin pedírselo.

Seguridad y detección de anomalías

Alerta inmediata tras 3+ intentos de login fallidos en 5 minutos. Detección de conexiones desde IPs en países no configurados con geolocalización automática. Alerta por equipos sin sincronizar más de N días (default: 5). Software no autorizado detectado en tiempo real. Audit logs con retención mínima de 7 años.

Despliegue masivo y administración remota

Paquete MSI firmado desplegable vía GPO, SCCM, Intune o PowerShell. Instalación silenciosa sin reinicio. Configuración centralizada desde el portal. Actualización remota de la flota completa con un clic: Cloud Run envía el comando vía Pub/Sub y el portal muestra el progreso en tiempo real.

📱 Analítica visual desde WhatsApp v1.1

Los usuarios autorizados consultan KPIs, alertas, estado de flota y reportes directamente desde WhatsApp, recibiendo gráficos generados en tiempo real. Gobierno completo de roles con 5 niveles (Super Admin, Admin TI, Gerente de Área, Analista, Viewer). Autenticación OTP vinculada al correo corporativo. Token JWT con expiración de 8h. Rate limiting (30 consultas/hora). Datos PII enmascarados automáticamente. Audit log de todas las interacciones.

🧠 Aprendizaje continuo a partir de errores sistémicos v1.1

El sistema cierra el ciclo de retroalimentación: cada alerta resuelta recibe clasificación TP/FP/FN del operador. Un detector identifica errores sistémicos (≥3 equipos con la misma falla en 1h) y genera informes prioritarios. Un optimizador semanal propone ajustes de umbrales con justificación estadística (requiere aprobación del Admin). Cada 15 días, el modelo de predicción de Vertex AI se reentrena con los nuevos datos de resolución, mejorando continuamente la precisión.

📐 Creación dinámica de indicadores de negocio (KPI Builder) v1.1

Los usuarios autorizados describen en lenguaje natural el indicador que necesitan (ej: "eficiencia del equipo de ventas") y un agente IA (AGT-14 KPI Advisor) sugiere fórmulas, metas y fuentes de datos relevantes a partir del esquema BigQuery. El usuario selecciona, personaliza y publica el KPI en el dashboard y/o en el canal WhatsApp. El cálculo se ejecuta de forma programada (batch) según la granularidad elegida.

Seguridad y privacidad
🔐 Seguridad en el endpoint
  • Logs locales cifrados AES-256-GCM con clave derivada del TPM del equipo
  • Directorio del agente con ACL restrictivo: solo SYSTEM y el servicio tienen acceso
  • Comunicación con GCP vía TLS 1.3; MTLS opcional en entornos de alta seguridad
  • El usuario local no puede leer ni modificar archivos del agente
  • Buffer offline inaccesible al usuario final en todo momento
☁️ Seguridad en GCP
  • IAM con mínimo privilegio: el agente solo tiene permiso pubsub.publish
  • Cloud Armor contra DDoS y OWASP Top 10 en el API del portal
  • VPC Service Controls aísla BigQuery y Spanner del acceso externo
  • Audit Logs habilitados en todos los servicios; retención mínima 7 años
  • Secret Manager para credenciales del agente, tokens y claves API
🛡️ Privacidad y cumplimiento
  • ActivityTracker captura solo título de ventana y dominio URL; nunca contenido de mensajes
  • Modo privacidad organizacional configurable por rol (RRHH, Legal, Dirección)
  • Anonimización de datos personales en BigQuery vía enmascaramiento de columnas
  • Cumplimiento con habeas data (Colombia), GDPR si aplica, y políticas corporativas
  • Datos NO accesibles al usuario final; solo a administradores autorizados
📱 Seguridad del Canal WhatsApp v1.1
  • ▸ Solo números registrados en eq_whatsapp_users pueden interactuar con el bot
  • ▸ Primera autenticación requiere OTP enviado al correo corporativo
  • ▸ Token JWT con TTL de 8 horas; después requiere re-autenticación
  • ▸ Rate limiting: máximo 30 consultas/hora por usuario
  • ▸ Datos PII enmascarados automáticamente (ej: "M. R***s")
  • ▸ Datos sensibles como mensajes temporales (view-once) cuando disponible
  • ▸ Productividad individual NO accesible vía WhatsApp (salvo Admin TI/Super Admin)
