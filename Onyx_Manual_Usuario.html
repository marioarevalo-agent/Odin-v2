📖 Manual de Usuario

Guía completa para administrar y operar la plataforma Onyx — Plataforma de Monitoreo Inteligente de activos tecnológicos.

Onyx v1.0 By Agentica Última actualización: Junio 2026

🔐 Acceso a la Plataforma

La plataforma Onyx está disponible desde cualquier navegador web moderno (Chrome, Edge, Firefox). No requiere instalación en el dispositivo del usuario.

URL de Acceso

https://proy-anla-poc-175647544738.us-central1.run.app

Inicio de Sesión

1

Ingresar credenciales

Ingresa tu email corporativo y contraseña proporcionados por el administrador.

2

Autenticación

El sistema valida tus credenciales y genera un token de sesión seguro (expira en 8 horas).

3

Acceso según rol

El dashboard se adapta automáticamente a tu rol (Administrador, Analista o Visor).

💡 Tu sesión permanece activa por 8 horas. Después de ese tiempo, deberás iniciar sesión nuevamente.

📊 Dashboard General

El dashboard principal muestra una vista consolidada del estado de toda la flota de equipos monitoreados en tiempo real.

Tarjetas KPI Principales

🖥️

Equipos Totales

Total de dispositivos registrados en la plataforma con el agente Onyx instalado.

🟢

Equipos Online

Dispositivos que han enviado datos en los últimos 10 minutos. Indica conectividad activa.

⚠️

Alertas Activas

Equipos con CPU >80%, RAM >85%, o disco <10GB disponible. Requieren atención.

📡

Latencia Promedio

Tiempo promedio de respuesta de red (ping a 8.8.8.8) de toda la flota.

💻 Vista por Equipo

Al hacer click en cualquier equipo de la tabla de flota, accedes a la vista detallada individual con toda la telemetría de ese dispositivo.

Información del Equipo

📋

Información General

ID del equipo, tipo de dispositivo, IP local, estado de batería (si es laptop), última sincronización.

🌐

Información de Red

IP local del equipo, latencia actual, estado de conexión.

Tarjetas de Estado

  • Tiempo Inactivo: Tiempo transcurrido sin actividad de teclado/mouse
  • Hora Pico: Hora del día con mayor actividad del usuario
  • Batería: Nivel de batería y estado de carga (solo laptops)
  • Última Actividad: Timestamp del último evento de actividad detectado

Gráficas Individuales

  • Uso de CPU (24h): Línea temporal del consumo de procesador
  • Uso de RAM (24h): Línea temporal del consumo de memoria
  • Latencia de Red (24h): Historial de tiempos de respuesta de red
  • Top Procesos: Los 5 procesos que más recursos consumen (CPU + RAM)

Detalle de Navegación

Muestra información de la conexión de red del equipo, incluyendo IP y estado de conectividad.

Causa Raíz

Cuando un equipo presenta alertas, el sistema identifica automáticamente la causa probable:

AlertaCausa RaízDetalle
CPU >80%High CPU usageProceso responsable (nombre, PID, % CPU)
RAM >85%High RAM usageProceso con mayor consumo de memoria
Disco <10GBLow disk spaceEspacio libre restante

📈 Productividad

La sección de productividad analiza el uso de aplicaciones y el tiempo activo de los equipos para generar métricas de eficiencia.

Métricas Principales

  • Horas Activas vs Inactivas: Distribución del tiempo de uso por equipo
  • Top Aplicaciones: Las aplicaciones más utilizadas por consumo de CPU/RAM
  • Tendencia Semanal: Gráfica de actividad a lo largo de la semana
  • Score de Productividad: Índice calculado basado en el uso eficiente del equipo

🛡️ Seguridad

Panel de monitoreo de eventos de seguridad y estado de protección de los endpoints.

Indicadores

  • Eventos de Seguridad: Registro de eventos relacionados con la seguridad del equipo
  • Estado de Antivirus: Verificación del estado de protección
  • Actualizaciones Pendientes: Equipos con actualizaciones de sistema operativo pendientes
⚠️ Los eventos de seguridad se registran automáticamente y no pueden ser eliminados. Constituyen un registro de auditoría inmutable.

🎫 Mesa de Ayuda

Sistema de tickets para gestión de incidencias y solicitudes de soporte técnico integrado con WhatsApp.

Funcionalidades

  • Creación de Tickets: Los usuarios pueden reportar problemas directamente
  • Asistente IA: Chat integrado con asistente inteligente que responde consultas técnicas
  • Resolución Automática: El sistema sugiere soluciones basadas en la telemetría del equipo
  • Métricas de Servicio: MTTR (tiempo medio de resolución), tickets por canal, satisfacción

🎯 KPI Builder

Herramienta para crear y monitorear indicadores de desempeño personalizados.

Crear un KPI

1

Definir nombre del KPI

Ejemplo: "Equipos con CPU crítica", "Tasa de conexión", "Latencia promedio"

2

Configurar fórmula

Utiliza fórmulas SQL sobre los datos de BigQuery. Ej: AVG(cpu_usage), COUNT(*)

3

Establecer objetivo

Define el valor target del KPI. El sistema mostrará el progreso con una barra visual.

✅ Los KPIs se calculan en tiempo real sobre los datos más recientes de BigQuery.

⬇️ Instalación del Agente Onyx

El agente Onyx se instala en cada equipo Windows que se desea monitorear. Recolecta métricas de hardware y las envía a la nube cada 5 minutos.

Requisitos

  • Sistema Operativo: Windows 10 / 11
  • Python: 3.10 o superior instalado
  • Permisos: Administrador local
  • Red: Conexión a Internet (soporta modo offline temporal)

Pasos de Instalación

1

Descargar el instalador

Desde la plataforma → Gestión Usuarios → botón "⬇️ Descargar Agente". Se descarga OnyxMonitor_Setup.zip.

2

Extraer el ZIP

Extraer la carpeta OnyxMonitor en la ubicación deseada (recomendado: C:\Onyx\agent\).

3

Ejecutar el instalador

Click derecho en install_service.bat → "Ejecutar como administrador".

4

Verificar

Abrir services.msc y confirmar que el servicio "OnyxMonitor" aparece como En ejecución.

💡 El instalador detecta automáticamente si hay una versión anterior (Tarea Programada) y la desactiva sin conflictos.

⚙️ Servicio Windows (OnyxMonitor)

A partir de la versión 2.1, el agente se ejecuta como un servicio de Windows nativo con auto-recuperación.

Características del Servicio

CaracterísticaValor
NombreOnyxMonitor
Tipo de inicioAutomático (delayed start)
CuentaLocalSystem
Auto-recoveryReinicio tras 60 segundos de fallo
HeartbeatCada 30 segundos (archivo + HTTP)
Intervalo de recolecciónCada 2 minutos

Comandos Útiles

# Ver estado del servicio
sc query OnyxMonitor

# Detener el servicio
net stop OnyxMonitor

# Iniciar el servicio
net start OnyxMonitor

# Reiniciar
net stop OnyxMonitor && net start OnyxMonitor

# Ver logs del servicio
type C:\Onyx\agent\onyx_service.log

Desinstalar

Para desinstalar el agente, ejecuta uninstall_service.bat como administrador, o desde Panel de Control → Agregar/Quitar Programas → Onyx Monitor.

🔧 Solución de Problemas

El equipo aparece como "Offline"

⚠️
Causas posibles:
  • El servicio OnyxMonitor no está ejecutándose
  • El equipo está apagado o sin conexión a Internet
  • Las credenciales de BigQuery expiraron
Solución:
  1. Verificar en services.msc que "OnyxMonitor" está Running
  2. Si está detenido, iniciarlo manualmente: net start OnyxMonitor
  3. Revisar el log: type C:\Onyx\agent\onyx_service.log

No puedo iniciar sesión

  • Verifica que el email esté escrito correctamente (sensible a mayúsculas)
  • Confirma que tu cuenta esté activa con el administrador
  • Si olvidaste tu contraseña, solicita un reset al administrador

Los datos no se actualizan

  • Presiona el botón 🔄 Actualizar en la esquina superior derecha
  • Los datos se actualizan automáticamente cada 2 minutos
  • Si persiste, verifica la conexión a Internet del equipo monitoreado

El instalador no funciona

  • Asegúrate de ejecutar install_service.bat como Administrador
  • Verifica que Python 3.10+ esté instalado y en el PATH
  • Revisa que el firewall permita conexiones HTTPS salientes
